METHOD 1: cPANEL FILE MANAGER (Easiest - No Software Needed)

Prerequisites

  • Hosting account with cPanel access
  • Your domain and hosting activated
  • Web browser (Chrome, Firefox, Edge)

Step 1: Access Your cPanel

1.1 Find Your cPanel Login Link

  • Check the email from your hosting provider
  • Look for: cPanel URL: yourdomain.com:2083
  • Write down your username and password

1.2 Login to cPanel

  • Open your browser
  • Type in address bar: yourdomain.com:2083
  • Or use: https://yourdomain.com/cpanel
  • Enter username and password
  • Click "Log In"

Step 2: Open File Manager

2.1 Click File Manager

  • Look for "File Manager" icon in cPanel
  • Click it
  • A window opens showing server folders

2.2 Navigate to public_html

  • On the left side, find folder list
  • Click on "public_html"
  • This is where all website files go

Step 3: Upload Your Files

Method A: Upload Individual Files

  1. In File Manager, click the "Upload" button
  2. Select files from: C:\xampp\htdocs\web
  3. Select each file and click "Open"
  4. Repeat for all files

Method B: Drag and Drop (Easiest)

  1. Open File Manager in one window (showing public_html)
  2. Open Windows File Explorer in another window
  3. Navigate to: C:\xampp\htdocs\web
  4. Drag files from Explorer → File Manager window
  5. Files upload automatically

Step 4: Set File Permissions

For Folders:

  1. Find your images folder
  2. Right-click on it
  3. Click "Change Permissions"
  4. Set to: 755
  5. Click "Change"

For PHP Files:

  1. Right-click send_contact.php
  2. Click "Change Permissions"
  3. Set to: 644
  4. Click "Change"

Step 5: Test Your Website

  1. Open browser
  2. Type: https://yourdomain.com
  3. Your website should load
  4. Test all links and features
✓ Success! Your website is now live using cPanel method.

METHOD 2: FTP CLIENT - FileZilla (Best for Large Sites)

Why Use FTP?

  • ✓ Faster uploads
  • ✓ Better for lots of files
  • ✓ Resume interrupted uploads
  • ✓ Drag-and-drop interface
  • ✓ Works on all hosting

Step 1: Download and Install FileZilla

1.1 Get FileZilla

  • Go to: https://filezilla-project.org/
  • Click "Download FileZilla Client"
  • Choose Windows (usually 64-bit)
  • Click download link

1.2 Install FileZilla

  • Open downloaded file: FileZilla_X.XX.X_win64.exe
  • Click "Next" → "Next" → "Install"
  • Click "Finish"
  • FileZilla opens

Step 2: Get FTP Credentials

2.1 Check Your Hosting Email

Look for email from hosting provider containing:

FTP Host: ftp.yourdomain.com Username: your_ftp_username Password: your_ftp_password Port: 21

2.2 Find in Hosting Control Panel

  • Login to hosting account
  • Look for "FTP Accounts" or "FTP Manager"
  • Click it
  • Write down the FTP details

Step 3: Add FTP Connection in FileZilla

3.1 Open Site Manager

  • In FileZilla, go to: File → Site Manager
  • Or press: Ctrl+S

3.2 Create New Site

  • Click "New Site"
  • Name it: MyWebsite

3.3 Enter FTP Details

Protocol: FTP - File Transfer Protocol Host: ftp.yourdomain.com Port: 21 User: your_ftp_username Password: your_ftp_password

3.4 Click "Connect"

  • FileZilla connects to your server
  • Wait 5-10 seconds
  • You should see: "Status: Connected"

Step 4: Navigate to public_html

4.1 After Connection

  • Look at RIGHT SIDE (Remote)
  • You might see: /home, /public_html
  • Double-click on public_html
  • Now in your website folder

Step 5: Prepare Local Folder

5.1 Navigate on Your Computer

  • Look at LEFT SIDE (Local)
  • Navigate to: C:\xampp\htdocs\web
  • You should see all website files

Step 6: Upload Files

Method A: Drag and Drop (Easiest)

  1. On LEFT side: Select all files (Ctrl+A)
  2. Drag them to RIGHT side (server)
  3. Watch progress bar
  4. Wait for 100% completion

Method B: Right-Click Upload

  1. Select files on LEFT side
  2. Right-click → "Upload"
  3. Files upload automatically

Step 7: Set Permissions

7.1 Set Folder Permissions to 755

  1. Right-click folder on RIGHT side (e.g., images)
  2. Click "File permissions"
  3. Enter: 755
  4. Click OK

7.2 Set File Permissions to 644

  1. Right-click send_contact.php
  2. Click "File permissions"
  3. Enter: 644
  4. Click OK

Step 8: Verify Upload

  1. Check RIGHT side for all files
  2. Open browser
  3. Type: https://yourdomain.com
  4. Website should load
✓ Success! Your website is now live using FTP method.

METHOD 3: GIT + COMMAND LINE (Professional Approach)

Why Use Git?

  • ✓ Version control (track changes)
  • ✓ Easy updates later
  • ✓ Collaborate with team
  • ✓ Backup your code
  • ✓ Professional workflow

Step 1: Install Git

1.1 Download Git

  • Go to: https://git-scm.com/download/win
  • Click download (usually Git-X.XX.X-64-bit.exe)
  • Open the downloaded file

1.2 Install Git

  • Click "Next" through installation
  • Choose options (defaults are fine)
  • Click "Finish"

1.3 Verify Installation

  • Open PowerShell
  • Type: git --version
  • Should show: git version 2.XX.X

Step 2: Create GitHub Account (Free)

2.1 Sign Up

  • Go to: https://github.com
  • Click "Sign up"
  • Create account with email and password

2.2 Create Personal Access Token

  1. Login to GitHub
  2. Click profile icon (top-right)
  3. Go to: Settings → Developer settings → Personal access tokens
  4. Click "Generate new token"
  5. Name it: my-deployment
  6. Check: repo (full control)
  7. Click "Generate token"
  8. COPY the token (save it somewhere safe)

Step 3: Create Repository on GitHub

3.1 Create New Repo

  1. Login to GitHub
  2. Click "+" icon (top-right)
  3. Click "New repository"
  4. Fill in:
    • Repository name: my-website
    • Description: My company website
    • Public/Private: Public
    • Leave other options unchecked
  5. Click "Create repository"

3.2 Copy Repository URL

  • You'll see: https://github.com/yourusername/my-website.git
  • Copy this URL (you'll need it)

Step 4: Initialize Git Repository

4.1 Open PowerShell

  • Right-click on desktop
  • Choose "Open PowerShell here"
  • Or: Open PowerShell and type: cd C:\xampp\htdocs\web

4.2 Initialize Git

git init

4.3 Configure Git (First Time Only)

git config --global user.name "Your Name" git config --global user.email "your-email@gmail.com"

Step 5: Add Your Files

5.1 Add All Files

git add .

5.2 Check Status

git status

5.3 Create First Commit

git commit -m "Initial upload - website files"

Step 6: Connect to GitHub

6.1 Add Remote Repository

git remote add origin https://github.com/yourusername/my-website.git

6.2 Verify Connection

git remote -v

Step 7: Push to GitHub

git branch -M main git push -u origin main

When prompted:

  • Username: Your GitHub username
  • Password: The Personal Access Token you created

Step 8: Deploy to Hosting Server

8.1 Connect via SSH

ssh username@your-hosting-server.com

8.2 Navigate to public_html

cd public_html

8.3 Clone Your Repository

git clone https://github.com/yourusername/my-website.git .

Step 9: Update Your Website Later

On Your Computer:

git add . git commit -m "Updated homepage" git push

On Your Hosting Server (via SSH):

git pull
✓ Success! Your website is now live using Git method.

TROUBLESHOOTING

❌ Website Won't Load
  • ✓ Check if files are in public_html
  • ✓ Verify permissions are set correctly
  • ✓ Check domain is connected to hosting
  • ✓ Wait 24-48 hours for DNS propagation
  • ✓ Clear browser cache (Ctrl+Shift+Delete)
❌ Contact Form Not Working
  • ✓ Update send_contact.php with SMTP settings
  • ✓ Verify email credentials are correct
  • ✓ Test with simple email first
  • ✓ Check email spam folder
❌ Permission Errors
  • ✓ Folders must be 755
  • ✓ Files must be 644
  • ✓ Re-check in File Manager
  • ✓ Use "Recursive" option for all subfolders
❌ Upload Stuck
  • ✓ Check internet connection
  • ✓ Try uploading fewer files at once
  • ✓ Use FTP instead of cPanel
  • ✓ Restart your router
  • ✓ Try different browser
ℹ️ Files Not Showing
  • ✓ Refresh the File Manager (F5)
  • ✓ Check if upload actually completed
  • ✓ Look in different folders
  • ✓ Check your internet didn't disconnect

RECOMMENDATIONS FOR BEGINNERS

✓ Easiest Path:
  1. Use Hostinger or Bluehost for hosting
  2. Use Method 1 (cPanel) for upload
  3. Configure SMTP for email
  4. Test everything thoroughly

  • Use Gmail (generate app password)
  • Or use Mailgun (free tier available)
  • Or use SendGrid (free tier available)
ℹ️ For AI Chatbot:
  • Use Gemini (free - best option)
  • Or use OpenAI (paid - better quality)
  • Or keep rule-based (no cost)

⚠️ IMPORTANT NOTES

  • Backup First: Always backup your files before uploading
  • Test Locally: Test website locally before deployment
  • SSL Certificate: Enable HTTPS/SSL for security
  • DNS Propagation: Domain activation may take 24-48 hours
  • File Permissions: Incorrect permissions cause 403 errors
  • PHP Version: Check hosting PHP version compatibility
  • Database: If using database, create it in hosting panel first
  • Support: Contact hosting support if issues arise

YOUR FILE STRUCTURE

Before upload, your local files should be organized as:

C:\xampp\htdocs\web\ ├── index.html ├── account.html ├── development.html ├── downloads.html ├── engineering.html ├── establishment.html ├── home.html ├── 404.html ├── style.css ├── script.js ├── translations.js ├── send_contact.php ├── images/ │ ├── logo.png │ └── [other images] └── PHPMailer-master/ ├── src/ └── language/

After upload, the same structure should be in your hosting public_html folder.
